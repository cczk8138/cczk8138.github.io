三、配置 pod库 工程环境变量

  1. dependency： 私有库依赖的三方库

  2. requires_arc: 指定私有库 文件是否 是ARC.默认是true,表示所有的 source_files是arc文件

注意：spec.requires_arc 指定的路径表示是arc文件，不被指定才会被标记 -fno-objc-arc

  1. frameworks: pod库使用的系统库

  2. weak_frameworks: 如果在高版本的 OS 中调用新增的功能，还要保持低版本 OS 能够运行，就要使用 weak_framwoks 如果引用的某些类或者接口在低版本中并不支持，可以再运行时判断。

  3. libraries 使用的静态库 比如 libz、sqlite3.0 等,多个用逗号分开

  4. compiler_flags: 传递个编译器的标记列表

  5. pod_target_xcconfig: flag 添加到私有库的 target Xcconfig 文件中，只会配置当前私有库

注意：尽量使用 pod_target_xcconfig，只会影响你编译的 pod

  1. prefix_header_file: 默认是 true 时 cocoapos 会生成默认前缀 .pch 文件

  2. prefix_header_contents 向pod项目的前缀文件中添加内容

  3. module_name

  4. header_dir

  5. header_mappings_dir


四、文件操作：podspec 文件必须在根仓库文件中。文件路径也是相对于根仓库位置的

模式1：*（检测文件名）

  • 匹配所有文件
    c 匹配以c开头的文件 c 匹配以c结尾的文件
    c 匹配以包括c的文件

模式2：**
递归匹配目录

模式3：？
匹配任何一个字符

模式4：[set]
匹配任何在set中的字符

模式5：{p, q}
匹配p，或q

模式6：遗弃下一个元字符
“JSONKit.?” #=> [“JSONKit.h”, “JSONKit.m”]
“.[a-z][a-z]” #=> [“CHANGELOG.md”, “README.md”]
“.[^m]” #=> [“JSONKit.h”]
“.{h,m}” #=> [“JSONKit.h”, “JSONKit.m”]
“*” #=> [“CHANGELOG.md”, “JSONKit.h”, “JSONKit.m”, “README.md”]

  1. source_files：pod文件路径

  2. public_header_files：公共头文件，这些头文件将暴露给用户的项目。如果不设置，所有source_files的头文件将被暴露

  1. private_header_files.和public_header_files相反，指定不暴露的头文件

  2. vendored_frameworks：使用的三方framework

  1. vendored libraries：三方静态库 指明具体路径

  2. resource_bundles：资源文件

  1. exclude_files:被排除的文件
